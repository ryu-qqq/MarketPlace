[[product-update-products-guide]]
==== 상품 일괄 수정 상세 가이드

===== 개요

상품 그룹 하위의 **옵션 구조**와 **상품(SKU)**을 한 번에 수정하는 API입니다.

====== 왜 옵션과 상품을 함께 보내야 하나?

상품(SKU)은 옵션 조합으로 결정됩니다. 예를 들어 "빨강-M" 티셔츠는 색상=빨강, 사이즈=M 옵션의 조합입니다.
옵션이 변경되면(추가/삭제) 상품도 함께 변경되어야 하므로, 두 데이터를 **원자적으로** 함께 보냅니다.


===== 핵심 개념: selectedOptions

====== 이름 기반 옵션 매핑

각 상품은 `selectedOptions` 배열로 자신이 속한 옵션 조합을 **이름으로 직접 지정**합니다.
옵션 그룹명(`optionGroupName`)과 옵션 값명(`optionValueName`)을 그대로 사용하므로 직관적입니다.

[source,json]
----
"selectedOptions": [
  { "optionGroupName": "색상", "optionValueName": "빨강" },
  { "optionGroupName": "사이즈", "optionValueName": "M" }
]
----

====== 예시

[source,json]
----
{
  "optionGroups": [
    {
      "optionGroupName": "색상",
      "optionValues": [
        { "optionValueName": "빨강", "sortOrder": 0 },
        { "optionValueName": "파랑", "sortOrder": 1 }
      ]
    },
    {
      "optionGroupName": "사이즈",
      "optionValues": [
        { "optionValueName": "S", "sortOrder": 0 },
        { "optionValueName": "M", "sortOrder": 1 },
        { "optionValueName": "L", "sortOrder": 2 }
      ]
    }
  ]
}
----

[cols="1,2,3", options="header"]
|===
| 상품 | 옵션 조합 | selectedOptions
| 빨강-S | 빨강 + S | `[{"optionGroupName":"색상","optionValueName":"빨강"},{"optionGroupName":"사이즈","optionValueName":"S"}]`
| 빨강-M | 빨강 + M | `[{"optionGroupName":"색상","optionValueName":"빨강"},{"optionGroupName":"사이즈","optionValueName":"M"}]`
| 빨강-L | 빨강 + L | `[{"optionGroupName":"색상","optionValueName":"빨강"},{"optionGroupName":"사이즈","optionValueName":"L"}]`
| 파랑-S | 파랑 + S | `[{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"S"}]`
| 파랑-M | 파랑 + M | `[{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"M"}]`
| 파랑-L | 파랑 + L | `[{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"L"}]`
|===


===== ID 규칙 (신규 vs 기존)

[cols="1,1,1", options="header"]
|===
| 필드 | null | non-null
| `sellerOptionGroupId` | 신규 옵션 그룹 추가 | 기존 옵션 그룹 수정
| `sellerOptionValueId` | 신규 옵션 값 추가 | 기존 옵션 값 유지/수정
| `productId` | 신규 상품 추가 | 기존 상품 수정
|===

IMPORTANT: 요청에 포함되지 않은 기존 ID는 **자동 삭제(soft delete)** 됩니다. 유지할 항목은 반드시 요청에 포함해야 합니다.

===== 내부 처리 흐름

[source]
----
프론트엔드 요청
  │
  ▼
1. 옵션 그룹 diff 수정 (SellerOptionCommandCoordinator)
   ├─ ID 있음 → 기존 옵션 수정
   ├─ ID null → 신규 옵션 생성 (DB에서 ID 할당)
   └─ 요청에 없는 기존 ID → soft delete
   │
   └─→ resolvedActiveValueIds 반환 (DB 확정 ID 포함)
  │
  ▼
2. 옵션 이름 → ID 매핑 구성
   └─→ optionGroups의 그룹명/값명 + resolvedActiveValueIds로 이름→ID 맵 생성
  │
  ▼
3. 상품(Product) diff 수정 (ProductCommandCoordinator)
   ├─ productId 있음 → 기존 상품 가격/재고/SKU/정렬 수정
   ├─ productId null → 신규 상품 생성 (selectedOptions 이름으로 옵션 매핑)
   └─ 요청에 없는 기존 productId → soft delete
----

**핵심**: 옵션이 먼저 DB에 반영된 후, `selectedOptions`의 이름을 사용해 실제 옵션 ID로 resolve합니다. 프론트엔드는 인덱스 계산 없이 이름만 전달하면 됩니다.


===== 프론트엔드 구현 가이드

====== selectedOptions 구성 (JavaScript)

[source,javascript]
----
function buildSelectedOptions(optionGroups, selectedValues) {
  // 각 옵션 그룹에서 선택한 값의 이름을 그대로 사용
  return selectedValues.map(selected => ({
    optionGroupName: selected.groupName,
    optionValueName: selected.valueName
  }));
}

// 예시: 색상(빨강,파랑) + 사이즈(S,M,L)에서 "파랑-M" 상품
// selectedValues = [{groupName: "색상", valueName: "파랑"}, {groupName: "사이즈", valueName: "M"}]
// → selectedOptions = [{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"M"}]
----

====== 옵션 변경 시 영향 없음

이름 기반 매핑이므로 옵션 값의 추가/삭제/순서 변경 시에도 **기존 상품의 `selectedOptions`를 재계산할 필요가 없습니다**:

* 옵션 값 추가 → 기존 상품 영향 없음
* 옵션 값 삭제 → 해당 옵션을 사용하는 상품만 제거
* 옵션 그룹 순서 변경 → 기존 상품 영향 없음

===== 주의사항

. **selectedOptions 이름 일치**: `optionGroupName`과 `optionValueName`은 `optionGroups`에 정의된 이름과 정확히 일치해야 합니다
. **상품별 selectedOptions 개수**: **전체 옵션 그룹 수**와 동일해야 합니다 (그룹 2개면 selectedOptions 2개, 그룹 1개면 selectedOptions 1개). 모든 그룹 포함
. **각 selectedOptions는 서로 다른 옵션 그룹에서 하나씩**: 같은 그룹의 옵션 값을 2개 선택하면 안 됩니다
. **삭제 = 제외**: 요청에 포함하지 않으면 자동 삭제됩니다. 유지할 항목은 반드시 포함
. **canonicalOptionGroupId/canonicalOptionValueId**: 선택사항. 표준 옵션에 매핑되지 않은 경우 null
. **inputType**: 생략 시 `PREDEFINED`(기본값). `FREE_INPUT` 그룹도 `optionValues`에 최소 1개 필요 (예: `"FREE"`, `"OS"`)
