[[product-group-register-guide]]
==== 상품 그룹 등록 상세 가이드

===== 개요

상품 그룹은 **옵션 구조(optionType)**에 따라 요청 형태가 달라집니다.
`optionType`은 상품 그룹의 옵션 전략을 결정하며, 이에 따라 `optionGroups`, `products`, `selectedOptions`의 구성 방식이 달라집니다.


===== optionType 종류

[cols="1,1,1,3", options="header"]
|===
| optionType | 이름 | 옵션 그룹 수 | 설명

| `COMBINATION` | 조합 옵션 | 2개 | 2개 옵션 그룹의 조합으로 SKU 결정 (예: 색상 x 사이즈)
| `SINGLE` | 단일 옵션 | 1개 | 1개 옵션 그룹으로 SKU 결정 (예: 사이즈만)
| `NONE` | 옵션 없음 | 0개 | 옵션 없이 상품 1개만 등록 (단품)
| `FREE_INPUT` | 자유 입력 | 0개 | 옵션 그룹 없이, 상품별로 자유롭게 SKU 구성
|===


===== 캐노니컬 옵션 (canonicalOptionGroupId / canonicalOptionValueId)

캐노니컬 옵션은 마켓플레이스 전체에서 공유하는 **표준 옵션 체계**입니다.
셀러가 직접 입력한 옵션("빨강", "레드", "RED")을 표준 옵션("Red")에 매핑하여, 검색/필터/비교 등에서 일관성을 확보합니다.

IMPORTANT: **캐노니컬 옵션 매핑은 선택사항입니다.** `canonicalOptionGroupId`와 `canonicalOptionValueId`는 `null`로 보내도 됩니다. 매핑하지 않아도 상품 등록/수정에 문제가 없습니다.

[cols="1,2,3", options="header"]
|===
| 필드 | null일 때 | non-null일 때

| `canonicalOptionGroupId` | 표준 옵션 그룹에 매핑하지 않음 | 표준 옵션 그룹에 매핑 (검색/필터 활용)
| `canonicalOptionValueId` | 표준 옵션 값에 매핑하지 않음 | 표준 옵션 값에 매핑 (비교/필터 활용)
|===

TIP: 캐노니컬 옵션 매핑 여부에 따라 상품의 동작은 달라지지 않습니다. 매핑은 **나중에 수정 API로 추가**할 수도 있습니다.


===== OptionType별 등록 예시

[[register-combination]]
====== 1. COMBINATION (조합 옵션) - 색상 x 사이즈

2개 옵션 그룹의 조합으로 상품이 결정됩니다. 가장 일반적인 패턴입니다.

[source,json]
----
{
  "sellerId": 1,
  "brandId": 100,
  "categoryId": 1000,
  "shippingPolicyId": 1,
  "refundPolicyId": 1,
  "productGroupName": "프리미엄 티셔츠",
  "optionType": "COMBINATION",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/tshirt_main.jpg", "sortOrder": 0 },
    { "imageType": "DETAIL", "originUrl": "https://example.com/tshirt_detail.jpg", "sortOrder": 1 }
  ],
  "optionGroups": [
    {
      "optionGroupName": "색상",
      "canonicalOptionGroupId": 1,
      "optionValues": [
        { "optionValueName": "빨강", "canonicalOptionValueId": 10, "sortOrder": 0 },
        { "optionValueName": "파랑", "canonicalOptionValueId": 11, "sortOrder": 1 }
      ]
    },
    {
      "optionGroupName": "사이즈",
      "canonicalOptionGroupId": 2,
      "optionValues": [
        { "optionValueName": "S", "canonicalOptionValueId": 20, "sortOrder": 0 },
        { "optionValueName": "M", "canonicalOptionValueId": 21, "sortOrder": 1 },
        { "optionValueName": "L", "canonicalOptionValueId": 22, "sortOrder": 2 }
      ]
    }
  ],
  "products": [
    { "skuCode": "RED-S",  "regularPrice": 30000, "currentPrice": 27000, "stockQuantity": 100, "sortOrder": 0, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"빨강"},{"optionGroupName":"사이즈","optionValueName":"S"}] },
    { "skuCode": "RED-M",  "regularPrice": 30000, "currentPrice": 27000, "stockQuantity": 80,  "sortOrder": 1, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"빨강"},{"optionGroupName":"사이즈","optionValueName":"M"}] },
    { "skuCode": "RED-L",  "regularPrice": 30000, "currentPrice": 27000, "stockQuantity": 60,  "sortOrder": 2, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"빨강"},{"optionGroupName":"사이즈","optionValueName":"L"}] },
    { "skuCode": "BLUE-S", "regularPrice": 32000, "currentPrice": 29000, "stockQuantity": 50,  "sortOrder": 3, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"S"}] },
    { "skuCode": "BLUE-M", "regularPrice": 32000, "currentPrice": 29000, "stockQuantity": 40,  "sortOrder": 4, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"M"}] },
    { "skuCode": "BLUE-L", "regularPrice": 32000, "currentPrice": 29000, "stockQuantity": 30,  "sortOrder": 5, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"파랑"},{"optionGroupName":"사이즈","optionValueName":"L"}] }
  ],
  "description": { "content": "<p>프리미엄 코튼 티셔츠</p>" },
  "notice": {
    "noticeCategoryId": 1,
    "entries": [
      { "noticeFieldId": 1, "fieldValue": "면 100%" },

    ]

----

IMPORTANT: `COMBINATION`은 반드시 **옵션 그룹 2개**가 필요합니다. 1개 또는 3개 이상이면 400 에러가 발생합니다.

**매핑 규칙**: 각 상품의 `selectedOptions`에는 옵션 그룹 2개에서 **각각 1개씩** 선택해야 합니다. 총 2개의 selectedOptions가 필요합니다.


[[register-single]]
====== 2. SINGLE (단일 옵션) - 사이즈만

1개 옵션 그룹으로 상품이 결정됩니다. 색상 없이 사이즈만 있는 경우 등에 사용합니다.

[source,json]
----
{
  "sellerId": 1,
  "brandId": 100,
  "categoryId": 1000,
  "shippingPolicyId": 1,
  "refundPolicyId": 1,
  "productGroupName": "기본 슬랙스",
  "optionType": "SINGLE",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/slacks_main.jpg", "sortOrder": 0 }
  ],
  "optionGroups": [
    {
      "optionGroupName": "사이즈",
      "canonicalOptionGroupId": 2,
      "optionValues": [
        { "optionValueName": "S",  "canonicalOptionValueId": 20, "sortOrder": 0 },
        { "optionValueName": "M",  "canonicalOptionValueId": 21, "sortOrder": 1 },
        { "optionValueName": "L",  "canonicalOptionValueId": 22, "sortOrder": 2 },
        { "optionValueName": "XL", "canonicalOptionValueId": 23, "sortOrder": 3 }
      ]
    }
  ],
  "products": [
    { "skuCode": "SLACKS-S",  "regularPrice": 45000, "currentPrice": 40000, "stockQuantity": 100, "sortOrder": 0, "selectedOptions": [{"optionGroupName":"사이즈","optionValueName":"S"}] },
    { "skuCode": "SLACKS-M",  "regularPrice": 45000, "currentPrice": 40000, "stockQuantity": 80,  "sortOrder": 1, "selectedOptions": [{"optionGroupName":"사이즈","optionValueName":"M"}] },
    { "skuCode": "SLACKS-L",  "regularPrice": 45000, "currentPrice": 40000, "stockQuantity": 60,  "sortOrder": 2, "selectedOptions": [{"optionGroupName":"사이즈","optionValueName":"L"}] },
    { "skuCode": "SLACKS-XL", "regularPrice": 47000, "currentPrice": 42000, "stockQuantity": 40,  "sortOrder": 3, "selectedOptions": [{"optionGroupName":"사이즈","optionValueName":"XL"}] }
  ],
  "description": { "content": "<p>데일리 슬랙스</p>" }
}
----

IMPORTANT: `SINGLE`은 반드시 **옵션 그룹 1개**가 필요합니다. 0개 또는 2개 이상이면 400 에러가 발생합니다.

**매핑 규칙**: 각 상품의 `selectedOptions`에는 **1개**만 있어야 합니다.

TIP: 캐노니컬 옵션 없이 등록하려면 `canonicalOptionGroupId`와 `canonicalOptionValueId`를 생략하거나 `null`로 보내면 됩니다.


[[register-none]]
====== 3. NONE (옵션 없음) - 단품

옵션 구조 없이 상품 1개만 등록합니다. 에어팟, 보조배터리 같은 단일 상품에 적합합니다.

[source,json]
----
{
  "sellerId": 1,
  "brandId": 100,
  "categoryId": 1000,
  "shippingPolicyId": 1,
  "refundPolicyId": 1,
  "productGroupName": "에어팟 프로 2세대",
  "optionType": "NONE",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/airpods_main.jpg", "sortOrder": 0 }
  ],
  "products": [
    { "skuCode": "AIRPODS-PRO-2", "regularPrice": 359000, "currentPrice": 329000, "stockQuantity": 200, "sortOrder": 0, "selectedOptions": [] }
  ],
  "description": { "content": "<p>Apple 에어팟 프로 2세대</p>" }
}
----

IMPORTANT: `NONE`에서는 `optionGroups`를 **보내지 않거나 빈 배열 `[]`** 이어야 합니다. 옵션 그룹을 보내면 400 에러가 발생합니다.

**매핑 규칙**: 상품의 `selectedOptions`는 **빈 배열 `[]`** 이어야 합니다.

TIP: `NONE` 타입은 보통 상품 1개만 등록하지만, 여러 개를 등록하는 것도 가능합니다 (예: 같은 상품의 일반/리퍼 구분 등).


[[register-free-input]]
====== 4. FREE_INPUT (자유 입력)

옵션 그룹을 정의하지 않고, 상품별로 **자유롭게 SKU를 구성**합니다.
옵션 그룹/값 체계를 사용하지 않으므로 `selectedOptions`는 빈 배열입니다.

대표적인 사용 사례:
* 주문 제작 상품 (커스텀 텍스트/이미지)
* 규격화되지 않은 빈티지/중고 상품
* 묶음 상품 (세트 A, 세트 B 등)

[source,json]
----
{
  "sellerId": 1,
  "brandId": 100,
  "categoryId": 1000,
  "shippingPolicyId": 1,
  "refundPolicyId": 1,
  "productGroupName": "커스텀 폰케이스",
  "optionType": "FREE_INPUT",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/phonecase_main.jpg", "sortOrder": 0 },
    { "imageType": "DETAIL", "originUrl": "https://example.com/phonecase_detail.jpg", "sortOrder": 1 }
  ],
  "products": [
    { "skuCode": "CASE-IP15-CLEAR",  "regularPrice": 25000, "currentPrice": 20000, "stockQuantity": 500, "sortOrder": 0, "selectedOptions": [] },
    { "skuCode": "CASE-IP15-MATTE",  "regularPrice": 28000, "currentPrice": 23000, "stockQuantity": 300, "sortOrder": 1, "selectedOptions": [] },
    { "skuCode": "CASE-IP14-CLEAR",  "regularPrice": 22000, "currentPrice": 18000, "stockQuantity": 400, "sortOrder": 2, "selectedOptions": [] }
  ],
  "description": { "content": "<p>맞춤 제작 폰케이스 - 주문 시 디자인 선택</p>" }
}
----

IMPORTANT: `FREE_INPUT`에서는 `optionGroups`를 **보내지 않거나 빈 배열 `[]`** 이어야 합니다. 옵션 그룹을 보내면 400 에러가 발생합니다.

**매핑 규칙**: 모든 상품의 `selectedOptions`는 **빈 배열 `[]`** 이어야 합니다.

WARNING: `FREE_INPUT`은 캐노니컬 옵션 매핑이 **불가능**합니다. 옵션 그룹 자체가 없으므로 `canonicalOptionGroupId`/`canonicalOptionValueId`를 설정할 수 없습니다. 캐노니컬 매핑을 통한 검색/필터가 필요하다면 `SINGLE` 또는 `COMBINATION`을 사용하세요.


===== FREE_INPUT vs NONE 비교

[cols="2,3,3", options="header"]
|===
| 구분 | NONE | FREE_INPUT

| 의도 | 옵션이 진짜 없는 단품 | 옵션 체계 없이 자유롭게 SKU 구성
| 상품 수 | 보통 1개 | 여러 개 (자유)
| SKU 예시 | `AIRPODS-PRO-2` | `CASE-IP15-CLEAR`, `CASE-IP15-MATTE`, ...
| 캐노니컬 매핑 | 불가 (옵션 없음) | 불가 (옵션 없음)
| 사용 사례 | 에어팟, 보조배터리 | 커스텀 제작, 세트 상품, 비규격 상품
|===


===== optionType별 요약 비교

[cols="1,1,1,1,1", options="header"]
|===
| | `COMBINATION` | `SINGLE` | `NONE` | `FREE_INPUT`

| 옵션 그룹 수 | 2개 필수 | 1개 필수 | 0개 (금지) | 0개 (금지)
| selectedOptions 크기 | 2 | 1 | 0 (빈 배열) | 0 (빈 배열)
| 캐노니컬 매핑 | 가능 (선택) | 가능 (선택) | 불가 | 불가
| 상품 수 제한 | 없음 | 없음 | 없음 | 없음
|===


===== 프론트엔드 구현 가이드

====== optionType에 따른 UI 분기

[source,javascript]
----
function buildRegisterRequest(optionType, optionGroups, products) {
  const request = {
    // ... 기본 필드 (sellerId, brandId 등)
    optionType: optionType
  };

  switch (optionType) {
    case 'COMBINATION':
      // 옵션 그룹 2개 + 각 상품에 selectedOptions 2개씩
      request.optionGroups = optionGroups; // 2개
      request.products = products.map(p => ({
        ...p,
        selectedOptions: p.selectedOptions // [{groupName, valueName}, {groupName, valueName}]
      }));
      break;

    case 'SINGLE':
      // 옵션 그룹 1개 + 각 상품에 selectedOptions 1개씩
      request.optionGroups = optionGroups; // 1개
      request.products = products.map(p => ({
        ...p,
        selectedOptions: p.selectedOptions // [{groupName, valueName}]
      }));
      break;

    case 'NONE':
    case 'FREE_INPUT':
      // 옵션 그룹 없음 + 각 상품에 selectedOptions 빈 배열
      // optionGroups를 보내지 않거나 빈 배열
      request.products = products.map(p => ({
        ...p,
        selectedOptions: [] // 항상 빈 배열
      }));
      break;
  }

  return request;
}
----

====== 캐노니컬 옵션 매핑 (선택사항)

캐노니컬 옵션은 `SINGLE`, `COMBINATION`에서만 사용 가능합니다.
매핑 여부에 따라 필드만 추가하면 됩니다:

[source,javascript]
----
// 캐노니컬 매핑 없이 (항상 가능)
{ "optionGroupName": "색상", "optionValues": [{ "optionValueName": "빨강", "sortOrder": 0 }] }

// 캐노니컬 매핑 포함 (선택)
{ "optionGroupName": "색상", "canonicalOptionGroupId": 1,
  "optionValues": [{ "optionValueName": "빨강", "canonicalOptionValueId": 10, "sortOrder": 0 }] }
----

TIP: 캐노니컬 옵션 ID는 `/canonical-option-groups` API에서 조회할 수 있습니다.


===== 주의사항

. **optionType과 optionGroups 수 일치 필수**: `COMBINATION`=2개, `SINGLE`=1개, `NONE`/`FREE_INPUT`=0개. 불일치 시 400 에러
. **selectedOptions 개수 = 옵션 그룹 수**: `COMBINATION`은 2개, `SINGLE`은 1개, `NONE`/`FREE_INPUT`은 빈 배열
. **selectedOptions 이름 일치**: `optionGroupName`과 `optionValueName`은 `optionGroups`에 정의된 이름과 정확히 일치해야 합니다
. **캐노니컬 옵션은 선택**: `canonicalOptionGroupId`/`canonicalOptionValueId`는 null 허용. 나중에 수정 API로 추가 가능
. **FREE_INPUT은 캐노니컬 매핑 불가**: 옵션 그룹 자체가 없으므로 캐노니컬 체계 사용 불가. 검색/필터 필요 시 `SINGLE`/`COMBINATION` 사용
