==== 상품 그룹 등록 상세 가이드

===== 개요

상품 그룹은 **옵션 구조(optionType)**에 따라 요청 형태가 달라집니다.
`optionType`은 상품 그룹의 옵션 전략을 결정하며, 이에 따라 `optionGroups`, `products`, `selectedOptions`의 구성 방식이 달라집니다.


===== optionType 종류

[cols="1,1,1,3", options="header"]
|===
| optionType | 이름 | 전체 그룹 수 | 설명

| `COMBINATION` | 조합 옵션 | **정확히 2개** | 2개 옵션 그룹의 조합으로 SKU 결정 (예: 색상 x 사이즈)
| `SINGLE` | 단일 옵션 | **정확히 1개** | 1개 옵션 그룹으로 SKU 결정 (예: 사이즈만)
| `NONE` | 옵션 없음 | **0개** | 옵션 없이 상품 등록 (단품)
|===


===== 캐노니컬 옵션 (canonicalOptionGroupId / canonicalOptionValueId)

캐노니컬 옵션은 마켓플레이스 전체에서 공유하는 **표준 옵션 체계**입니다.
셀러가 직접 입력한 옵션("빨강", "레드", "RED")을 표준 옵션("Red")에 매핑하여, 검색/필터/비교 등에서 일관성을 확보합니다.

IMPORTANT: **캐노니컬 옵션 매핑은 선택사항입니다.** `canonicalOptionGroupId`와 `canonicalOptionValueId`는 `null`로 보내도 됩니다. 매핑하지 않아도 상품 등록/수정에 문제가 없습니다.

[cols="1,2,3", options="header"]
|===
| 필드 | null일 때 | non-null일 때

| `canonicalOptionGroupId` | 표준 옵션 그룹에 매핑하지 않음 | 표준 옵션 그룹에 매핑 (검색/필터 활용)
| `canonicalOptionValueId` | 표준 옵션 값에 매핑하지 않음 | 표준 옵션 값에 매핑 (비교/필터 활용)
|===

TIP: 캐노니컬 옵션 매핑 여부에 따라 상품의 동작은 달라지지 않습니다. 매핑은 **나중에 수정 API로 추가**할 수도 있습니다.


===== inputType (옵션 그룹 입력 유형)

`SINGLE`, `COMBINATION` 옵션타입에서 각 옵션 그룹은 `inputType` 필드로 **입력 방식**을 지정할 수 있습니다.

[cols="1,2,3", options="header"]
|===
| inputType | 이름 | 설명

| `PREDEFINED` | 사전 정의 (기본값) | 셀러가 미리 정의한 옵션 값 목록에서 선택 (예: 색상, 사이즈).
| `FREE_INPUT` | 자유 입력 | OMS 연동 시 외부 몰에 자유 입력 옵션으로 매핑됨을 표시 (예: 각인 문구). `optionValues`에 최소 1개 필요 (예: `"OS"`, `"FREE"`).
|===

IMPORTANT: 모든 `inputType`에서 `optionValues`는 **최소 1개 이상** 필요합니다. 빈 배열은 허용되지 않습니다.

IMPORTANT: `inputType`을 생략하면 기본값 `"PREDEFINED"`가 적용됩니다.


===== optionType × 그룹 수 규칙

`optionType`에 따라 전체 옵션 그룹 수가 고정됩니다. `inputType`은 그룹 수에 영향을 주지 않습니다.

[cols="2,2,1,3", options="header"]
|===
| optionType | 전체 그룹 수 | 결과 | 사용 사례

| `COMBINATION` | 2개 | ✅ | 색상 x 사이즈, PREDEFINED 2개 또는 PREDEFINED + FREE_INPUT
| `COMBINATION` | 0~1개 또는 3개+ | ❌ 400 | 정확히 2개 필수
| `SINGLE` | 1개 | ✅ | 사이즈만 (PREDEFINED) 또는 각인 전용 (FREE_INPUT)
| `SINGLE` | 0개 또는 2개+ | ❌ 400 | 정확히 1개 필수
| `NONE` | 0개 | ✅ | 에어팟 (단품)
| `NONE` | 1개+ | ❌ 400 | 그룹 존재 시 에러
|===

IMPORTANT: `selectedOptions`는 모든 옵션 그룹을 포함해야 합니다. `COMBINATION`=2개, `SINGLE`=1개, `NONE`=0개.


===== OptionType별 등록 예시

[[register-combination]]
====== 1. COMBINATION — PREDEFINED만 (색상 x 사이즈)

2개 PREDEFINED 옵션 그룹의 조합으로 SKU가 결정됩니다. 가장 일반적인 패턴입니다.

      "optionGroupName": "색상", "inputType": "PREDEFINED",
      "optionValues": [
        { "optionValueName": "빨강", "sortOrder": 0 },

        { "optionValueName": "블랙", "sortOrder": 1 }

    {
      "optionGroupName": "각인 문구", "inputType": "FREE_INPUT",
      "optionValues": [
        { "optionValueName": "FREE", "sortOrder": 0 }

  ],
  "products": [
    { "skuCode": "CUST-W", "regularPrice": 35000, "currentPrice": 30000, "stockQuantity": 50, "sortOrder": 0, "selectedOptions": [{"optionGroupName":"색상","optionValueName":"화이트"},{"optionGroupName":"각인 문구","optionValueName":"FREE"}] },

IMPORTANT: `selectedOptions`에는 **모든 옵션 그룹**을 포함합니다. FREE_INPUT 그룹도 반드시 `optionValues` 1개 이상이 필요하고, `selectedOptions`에 포함되어야 합니다.

====== 4. SINGLE — FREE_INPUT (각인 전용)

FREE_INPUT 그룹 1개로 구성합니다. 커스텀 제작, 각인 전용 상품에 적합합니다.

[source,json]
----
{
  "sellerId": 1, "brandId": 100, "categoryId": 1000,
  "shippingPolicyId": 1, "refundPolicyId": 1,
  "productGroupName": "각인 전용 반지",
  "optionType": "SINGLE",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/ring_engrave.jpg", "sortOrder": 0 }
  ],
  "optionGroups": [
    {
      "optionGroupName": "각인 문구", "inputType": "FREE_INPUT",
      "optionValues": [
        { "optionValueName": "FREE", "sortOrder": 0 }
      ]
    }
  ],
  "products": [
    { "skuCode": "ENGRAVE-RING-001", "regularPrice": 89000, "currentPrice": 79000, "stockQuantity": 100, "sortOrder": 0, "selectedOptions": [{"optionGroupName":"각인 문구","optionValueName":"FREE"}] }
  ],
  "description": { "content": "<p>각인 전용 반지 - 주문 시 문구 입력</p>" }
}
----

IMPORTANT: FREE_INPUT 그룹도 `optionValues` **최소 1개** 필요합니다 (예: `"FREE"`, `"OS"`). `selectedOptions`에 해당 그룹을 포함해야 합니다.

[[register-none]]
====== 6. NONE — 옵션 없음 (단품)

옵션 구조 없이 상품만 등록합니다. 에어팟, 보조배터리 같은 단일 상품에 적합합니다.

[source,json]
----
{
  "sellerId": 1, "brandId": 100, "categoryId": 1000,
  "shippingPolicyId": 1, "refundPolicyId": 1,
  "productGroupName": "에어팟 프로 2세대",
  "optionType": "NONE",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/airpods_main.jpg", "sortOrder": 0 }
  ],
  "products": [
    { "skuCode": "AIRPODS-PRO-2", "regularPrice": 359000, "currentPrice": 329000, "stockQuantity": 200, "sortOrder": 0, "selectedOptions": [] }
  ],
  "description": { "content": "<p>Apple 에어팟 프로 2세대</p>" }
}
----

IMPORTANT: `NONE`에서는 `optionGroups`가 **빈 배열 `[]`** 이거나 생략해야 합니다. 어떤 그룹(PREDEFINED든 FREE_INPUT이든)이라도 보내면 400 에러가 발생합니다.

**매핑 규칙**: 상품의 `selectedOptions`는 **빈 배열 `[]`** 이어야 합니다.


===== optionType별 요약 비교

[cols="1,1,1,1", options="header"]
|===
| | `COMBINATION` | `SINGLE` | `NONE`

| 전체 그룹 수 | **정확히 2개** | **정확히 1개** | **0개**
| inputType 조합 | PREDEFINED 2개, PREDEFINED+FREE_INPUT, FREE_INPUT 2개 모두 가능 | PREDEFINED 또는 FREE_INPUT | 해당 없음
| optionValues | 모든 그룹에서 최소 1개 필수 | 최소 1개 필수 | 해당 없음
| selectedOptions 크기 | 2개 (모든 그룹 포함) | 1개 (모든 그룹 포함) | 0 (빈 배열)
| 캐노니컬 매핑 | 가능 (선택) | 가능 (선택) | 불가
| 상품 수 제한 | 없음 | 없음 | 없음
|===


===== 프론트엔드 구현 가이드

====== optionType에 따른 UI 분기

[source,javascript]
----
function buildRegisterRequest(optionType, optionGroups, products) {
  const request = {
    // ... 기본 필드 (sellerId, brandId 등)
    optionType: optionType
  };

  switch (optionType) {
    case 'COMBINATION':
      // 정확히 2개 그룹 필요 (inputType 무관)
      if (optionGroups.length !== 2) throw new Error('COMBINATION은 정확히 2개 그룹 필요');
      request.optionGroups = optionGroups.map(g => ({
        ...g,
        inputType: g.inputType || 'PREDEFINED'
      }));
      // selectedOptions: 모든 그룹에서 각 1개씩, 총 2개
      request.products = products.map(p => ({
        ...p,
        selectedOptions: p.selectedOptions // 모든 그룹에서 각 1개씩
      }));
      break;

    case 'SINGLE':
      // 정확히 1개 그룹 필요 (inputType 무관)
      if (optionGroups.length !== 1) throw new Error('SINGLE은 정확히 1개 그룹 필요');
      request.optionGroups = optionGroups.map(g => ({
        ...g,
        inputType: g.inputType || 'PREDEFINED'
      }));
      // selectedOptions: 해당 그룹에서 1개
      request.products = products.map(p => ({
        ...p,
        selectedOptions: p.selectedOptions // 1개
      }));
      break;

    case 'NONE':
      // 옵션 그룹 0개 + 각 상품에 selectedOptions 빈 배열
      if (optionGroups.length !== 0) throw new Error('NONE은 그룹 없어야 함');
      request.products = products.map(p => ({
        ...p,
        selectedOptions: [] // 항상 빈 배열
      }));
      break;
  }

  return request;
}
----

====== 캐노니컬 옵션 매핑 (선택사항)

캐노니컬 옵션은 `SINGLE`, `COMBINATION`에서만 사용 가능합니다.
매핑 여부에 따라 필드만 추가하면 됩니다:

[source,javascript]
----
// 기본 PREDEFINED 그룹 (inputType 생략 가능 - 기본값 PREDEFINED)
{ "optionGroupName": "색상", "optionValues": [{ "optionValueName": "빨강", "sortOrder": 0 }] }

// 캐노니컬 매핑 포함
{ "optionGroupName": "색상", "canonicalOptionGroupId": 1, "inputType": "PREDEFINED",
  "optionValues": [{ "optionValueName": "빨강", "canonicalOptionValueId": 10, "sortOrder": 0 }] }

// 자유 입력 그룹 (OMS 연동 시 외부 몰에 자유 입력으로 매핑)
{ "optionGroupName": "각인 문구", "inputType": "FREE_INPUT", "optionValues": [{ "optionValueName": "FREE", "sortOrder": 0 }] }
----

TIP: 캐노니컬 옵션 ID는 `/canonical-option-groups` API에서 조회할 수 있습니다.


===== 주의사항

. **optionType과 전체 그룹 수 일치 필수**: `COMBINATION`=정확히 2개, `SINGLE`=정확히 1개, `NONE`=0개. 불일치 시 400 에러
. **모든 그룹에 optionValues 최소 1개 필수**: `PREDEFINED`든 `FREE_INPUT`이든 `optionValues`가 빈 배열이면 400 에러
. **selectedOptions에는 모든 그룹 포함**: `COMBINATION`=2개, `SINGLE`=1개, `NONE`=0개. 모든 옵션 그룹에서 각각 1개씩 선택
. **selectedOptions 이름 일치**: `optionGroupName`과 `optionValueName`은 `optionGroups`에 정의된 이름과 정확히 일치해야 합니다
. **캐노니컬 옵션은 선택**: `canonicalOptionGroupId`/`canonicalOptionValueId`는 null 허용. 나중에 수정 API로 추가 가능
. **inputType 기본값**: `inputType`을 생략하면 `"PREDEFINED"`로 간주합니다
