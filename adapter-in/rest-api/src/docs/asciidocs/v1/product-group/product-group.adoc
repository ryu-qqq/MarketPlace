==== 상품 그룹 등록 상세 가이드

===== 개요

상품 그룹은 **옵션 구조(optionType)**에 따라 요청 형태가 달라집니다.
`optionType`은 상품 그룹의 옵션 전략을 결정하며, 이에 따라 `optionGroups`, `products`, `selectedOptions`의 구성 방식이 달라집니다.


===== optionType 종류

[cols="1,1,1,3", options="header"]
|===
| optionType | 이름 | 옵션 그룹 수 | 설명

| `COMBINATION` | 조합 옵션 | PREDEFINED 2개 | 2개 옵션 그룹의 조합으로 SKU 결정 (예: 색상 x 사이즈)
| `SINGLE` | 단일 옵션 | PREDEFINED 0~1개, 전체 1개 이상 | PREDEFINED 1개 또는 FREE_INPUT만으로 구성 가능 (예: 사이즈만, 각인 전용)
| `NONE` | 옵션 없음 | 0개 | 옵션 없이 상품 등록 (단품). 프리인풋 포함 불가
|===


===== 캐노니컬 옵션 (canonicalOptionGroupId / canonicalOptionValueId)

캐노니컬 옵션은 마켓플레이스 전체에서 공유하는 **표준 옵션 체계**입니다.
셀러가 직접 입력한 옵션("빨강", "레드", "RED")을 표준 옵션("Red")에 매핑하여, 검색/필터/비교 등에서 일관성을 확보합니다.

IMPORTANT: **캐노니컬 옵션 매핑은 선택사항입니다.** `canonicalOptionGroupId`와 `canonicalOptionValueId`는 `null`로 보내도 됩니다. 매핑하지 않아도 상품 등록/수정에 문제가 없습니다.

[cols="1,2,3", options="header"]
|===
| 필드 | null일 때 | non-null일 때

| `canonicalOptionGroupId` | 표준 옵션 그룹에 매핑하지 않음 | 표준 옵션 그룹에 매핑 (검색/필터 활용)
| `canonicalOptionValueId` | 표준 옵션 값에 매핑하지 않음 | 표준 옵션 값에 매핑 (비교/필터 활용)
|===

TIP: 캐노니컬 옵션 매핑 여부에 따라 상품의 동작은 달라지지 않습니다. 매핑은 **나중에 수정 API로 추가**할 수도 있습니다.


===== inputType (옵션 그룹 입력 유형)

`SINGLE`, `COMBINATION` 옵션타입에서 각 옵션 그룹은 `inputType` 필드로 **입력 방식**을 지정할 수 있습니다.

[cols="1,2,3", options="header"]
|===
| inputType | 이름 | 설명

| `PREDEFINED` | 사전 정의 (기본값) | 셀러가 미리 정의한 옵션 값 목록에서 선택 (예: 색상, 사이즈). **SKU 조합에 참여**.
| `FREE_INPUT` | 자유 입력 | 구매자가 주문 시 직접 입력 (예: 각인 문구, 커스텀 텍스트). **SKU 조합에 참여하지 않음**. `optionValues`는 빈 배열.
|===

IMPORTANT: `inputType`을 생략하면 기본값 `"PREDEFINED"`가 적용됩니다. 기존 API와 완전히 **하위호환**됩니다.

TIP: **혼합 모드** 지원 — 하나의 상품그룹에 `PREDEFINED` 그룹과 `FREE_INPUT` 그룹을 함께 사용할 수 있습니다. 예를 들어 반지 상품에서 사이즈(PREDEFINED)와 각인 문구(FREE_INPUT)를 동시에 제공할 수 있습니다.


===== optionType × inputType 조합 매트릭스

아래 표는 가능한 모든 조합을 보여줍니다. 각 조합의 상세 예시는 아래에서 확인하세요.

[cols="2,2,2,1,3", options="header"]
|===
| optionType | PREDEFINED 그룹 | FREE_INPUT 그룹 | 결과 | 사용 사례

| `COMBINATION` | 2개 | 0개 | ✅ | 색상 x 사이즈
| `COMBINATION` | 2개 | 1개+ | ✅ | 색상 x 사이즈 + 프린팅 문구
| `COMBINATION` | 0~1개 | any | ❌ 400 | PREDEFINED 2개 필수
| `SINGLE` | 1개 | 0개 | ✅ | 사이즈만
| `SINGLE` | 1개 | 1개+ | ✅ | 사이즈 + 각인 문구
| `SINGLE` | 0개 | 1개+ | ✅ | 각인 전용 (커스텀 제작)
| `SINGLE` | 2개+ | any | ❌ 400 | PREDEFINED 1개 초과 불가
| `SINGLE` | 0개 | 0개 | ❌ 400 | 전체 그룹 1개 이상 필수
| `NONE` | 0개 | 0개 | ✅ | 에어팟 (단품)
| `NONE` | any | any | ❌ 400 | 그룹 존재 시 에러
|===

옵션 구조 없이 상품만 등록합니다. 에어팟, 보조배터리 같은 단일 상품에 적합합니다.

[source,json]
----
{
  "sellerId": 1, "brandId": 100, "categoryId": 1000,
  "shippingPolicyId": 1, "refundPolicyId": 1,
  "productGroupName": "에어팟 프로 2세대",
  "optionType": "NONE",
  "images": [
    { "imageType": "THUMBNAIL", "originUrl": "https://example.com/airpods_main.jpg", "sortOrder": 0 }
  ],
  "products": [
    { "skuCode": "AIRPODS-PRO-2", "regularPrice": 359000, "currentPrice": 329000, "stockQuantity": 200, "sortOrder": 0, "selectedOptions": [] }
  ],
  "description": { "content": "<p>Apple 에어팟 프로 2세대</p>" }
}
----

IMPORTANT: `NONE`에서는 `optionGroups`가 **빈 배열 `[]`** 이거나 생략해야 합니다. 어떤 그룹(PREDEFINED든 FREE_INPUT이든)이라도 보내면 400 에러가 발생합니다.

**매핑 규칙**: 상품의 `selectedOptions`는 **빈 배열 `[]`** 이어야 합니다.


===== optionType별 요약 비교

[cols="1,1,1,1", options="header"]
|===
| | `COMBINATION` | `SINGLE` | `NONE`

| PREDEFINED 그룹 수 | 2개 필수 | 0~1개 | 0개 (금지)
| 전체 그룹 수 | 2개 이상 | 1개 이상 | 0개 (금지)
| FREE_INPUT 그룹 | 추가 가능 (선택) | 추가 가능 (선택) | 불가
| selectedOptions 크기 | 2 (PREDEFINED만) | 0~1 (PREDEFINED만) | 0 (빈 배열)
| 캐노니컬 매핑 | 가능 (선택) | 가능 (선택) | 불가
| 상품 수 제한 | 없음 | 없음 | 없음
|===


===== 프론트엔드 구현 가이드

====== optionType에 따른 UI 분기

[source,javascript]
----
function buildRegisterRequest(optionType, optionGroups, products) {
  const request = {
    // ... 기본 필드 (sellerId, brandId 등)
    optionType: optionType
  };

  switch (optionType) {
    case 'COMBINATION':
      // PREDEFINED 그룹 2개 (+ 선택적 FREE_INPUT 그룹)
      request.optionGroups = optionGroups.map(g => ({
        ...g,
        inputType: g.inputType || 'PREDEFINED' // 생략 시 기본값 PREDEFINED
      }));
      // selectedOptions: PREDEFINED 그룹에서만 선택
      request.products = products.map(p => ({
        ...p,
        selectedOptions: p.selectedOptions // PREDEFINED 그룹에서 각 1개씩, 총 2개
      }));
      break;

    case 'SINGLE':
      // PREDEFINED 0~1개 + 선택적 FREE_INPUT 그룹 (전체 1개 이상)
      request.optionGroups = optionGroups.map(g => ({
        ...g,
        inputType: g.inputType || 'PREDEFINED'
      }));
      // PREDEFINED 그룹이 있으면 selectedOptions에 1개, FREE_INPUT만이면 빈 배열
      const hasPredefined = optionGroups.some(g => (g.inputType || 'PREDEFINED') === 'PREDEFINED');
      request.products = products.map(p => ({
        ...p,
        selectedOptions: hasPredefined ? p.selectedOptions : [] // PREDEFINED 없으면 빈 배열
      }));
      break;

    case 'NONE':
      // 옵션 그룹 없음 + 각 상품에 selectedOptions 빈 배열
      // optionGroups를 보내지 않거나 빈 배열
      request.products = products.map(p => ({
        ...p,
        selectedOptions: [] // 항상 빈 배열
      }));
      break;
  }

  return request;
}
----

====== 캐노니컬 옵션 매핑 (선택사항)

캐노니컬 옵션은 `SINGLE`, `COMBINATION`에서만 사용 가능합니다.
매핑 여부에 따라 필드만 추가하면 됩니다:

[source,javascript]
----
// 기본 PREDEFINED 그룹 (inputType 생략 가능 - 기본값 PREDEFINED)
{ "optionGroupName": "색상", "optionValues": [{ "optionValueName": "빨강", "sortOrder": 0 }] }

// 캐노니컬 매핑 포함
{ "optionGroupName": "색상", "canonicalOptionGroupId": 1, "inputType": "PREDEFINED",
  "optionValues": [{ "optionValueName": "빨강", "canonicalOptionValueId": 10, "sortOrder": 0 }] }

// 자유 입력 그룹 (구매자가 주문 시 직접 입력)
{ "optionGroupName": "각인 문구", "inputType": "FREE_INPUT", "optionValues": [] }
----

TIP: 캐노니컬 옵션 ID는 `/canonical-option-groups` API에서 조회할 수 있습니다.


===== 주의사항

. **optionType과 PREDEFINED 그룹 수 일치 필수**: `COMBINATION`=PREDEFINED 2개, `SINGLE`=PREDEFINED 0~1개(전체 1개 이상), `NONE`=0개. 불일치 시 400 에러
. **FREE_INPUT 그룹은 개수 제한 없음**: `SINGLE`/`COMBINATION`에서 PREDEFINED 그룹 외에 FREE_INPUT 그룹을 자유롭게 추가 가능
. **FREE_INPUT 그룹의 optionValues는 빈 배열**: 구매자가 주문 시 직접 입력하므로 사전 정의 옵션 값이 없어야 합니다
. **selectedOptions에는 PREDEFINED 그룹만 포함**: FREE_INPUT 그룹은 SKU 조합에 참여하지 않으므로 `selectedOptions`에서 제외
. **selectedOptions 이름 일치**: `optionGroupName`과 `optionValueName`은 `optionGroups`에 정의된 이름과 정확히 일치해야 합니다
. **캐노니컬 옵션은 선택**: `canonicalOptionGroupId`/`canonicalOptionValueId`는 null 허용. 나중에 수정 API로 추가 가능
. **inputType 기본값**: `inputType`을 생략하면 `"PREDEFINED"`로 간주합니다. 기존 API 호출을 변경할 필요 없습니다
