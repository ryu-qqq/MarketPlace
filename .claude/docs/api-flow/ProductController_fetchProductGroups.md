## 6. 특이사항

**2-Phase Query 패턴**
Phase 1에서 WHERE 조건으로 ID만 조회하고, Phase 2에서 ID IN으로 상세 데이터를 JOIN 조회한다. 페이징 쿼리에서 JOIN으로 인한 페이징 오류를 방지하기 위한 패턴이다.

**권한 기반 필터링**
`@AuthorityValidate` AOP가 요청 전에 실행되어 SELLER 권한이면 `filter.sellerId`에 현재 로그인 판매자 ID를 자동으로 세팅한다. MASTER 권한이면 null로 두어 전체 조회가 된다.

**No-Offset 커서 페이징**
`lastDomainId`가 있으면 `productGroup.id.lt(lastDomainId)` 조건으로 No-Offset 방식을 사용한다. DESC 정렬이면 lt, ASC 정렬이면 gt를 적용한다.

**카테고리 경로 처리**
Phase 2 쿼리에서 `category.path`를 `"500,200,100"` 형태의 쉼표 구분 문자열로 가져온다. Service에서 이 값을 파싱하여 전체 카테고리 ID Set을 만들고 `CategoryFetchService.fetchAllCategories()`로 카테고리명을 일괄 조회한 뒤 `ProductGroupCategoryMapper`가 `categoryFullName`을 세팅한다.

**개별 상품 폴백 조회**
`setProducts()`에서 productGroupIds로 일괄 조회 후, 결과가 비어 있는 상품그룹은 해당 productGroupId로 개별 재조회한다. 이는 일부 상품그룹의 상품이 누락될 경우를 대비한 폴백 로직이다.

**GroupBy transform 패턴**
QueryDSL의 `transform(GroupBy.groupBy(...).list(...))` 를 사용하여 LEFT JOIN으로 발생하는 중복 행을 Java 메모리에서 집계한다. `externalProductInfos`는 `GroupBy.set()`으로 중복 없이 수집된다.
